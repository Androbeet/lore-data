==========================================
 LORE — UPLOAD CHECKLIST (the simple rule)
==========================================

RULE: UPLOAD EVERYTHING IN THIS FOLDER. ALL OF IT.

There are no "private" or "local-only" files here.
The docs/ guides are also uploaded — that's normal,
every professional repo ships its documentation.
Website visitors never see them; only index.html
is the website.

BEFORE UPLOADING:
[ ] Turn on hidden files so you can see them:
      Windows: File Explorer > View > Show > Hidden items
      Mac:     Finder, press Cmd + Shift + .

WHAT TO SELECT (everything):
[ ] index.html            <- the app
[ ] 404.html
[ ] .nojekyll             <- hidden file! easy to miss
[ ] manifest.webmanifest
[ ] sw.js
[ ] robots.txt
[ ] README.md
[ ] LICENSE
[ ] CONTRIBUTING.md
[ ] UPLOAD-CHECKLIST.txt  <- yes, even this :)
[ ] assets/   (logo + banner)
[ ] data/     (starter database files)
[ ] docs/     (all the guides)
[ ] worker/   (backend for later)
[ ] .github/  (hidden folder! robot scripts for later)

AFTER UPLOADING, CHECK:
[ ] index.html sits at the TOP LEVEL of the repo
    (not inside a "lore" subfolder)
[ ] .nojekyll is visible in the repo file list
    (if missing: Add file > Create new file > name it
     ".nojekyll" > commit empty)

THE ONLY THING YOU MUST NEVER UPLOAD (doesn't exist yet):
  X  Secret tokens / passwords. Later, when you go live,
     your GitHub token goes ONLY into Cloudflare's
     settings panel — never into any file in this repo.
